Portfolio Reflection – Data Structures and Algorithms

In this course, the main problem I was solving was how to organize and manage course data in a clear and efficient way. In Project One, I looked at different data structures and compared their run-time and memory usage. This helped me understand which structure works best in different situations. In Project Two, I wrote a program that reads course information, sorts it, and prints it in alphanumeric order. The goal was to make it easy for advisors and students to find course details quickly. Both projects focused on solving real problems using data structures and algorithms.

To solve these problems, I used a step-by-step approach. First, I studied how different data structures like vectors, hash tables, and binary search trees work. Then I chose the one that best fit the problem. Data structures are important because they affect how fast a program runs and how much memory it uses. While working on the projects, I faced some roadblocks, like errors in my code and trouble understanding how to organize the data correctly. I overcame these issues by testing my code often, breaking problems into smaller parts, and reviewing examples from class. Sometimes I had to rewrite parts of my code, but that helped me learn and improve.

This project changed the way I think about designing software. I now plan my programs more before I start coding. I think about what data structure I should use and how the program will grow in the future. It also helped me write cleaner and more readable code. I learned to use clear variable names and organize my code into functions so it is easier to understand. My programs are now more maintainable and adaptable because I can update or fix parts without breaking everything. Overall, this course helped me become more confident in solving problems and writing better programs.
